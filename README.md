## ✨ คุณสมบัติเด่น (Features)

* **⚡ AJAX Race Condition Avoidance**: ระบบตรวจเช็คสถานะ Dropdown แบบวนรอบสั้น (`get_time_dropdown`) ลดความหน่วงเวลาและสแกนคอร์ทเสร็จสิ้นในหลักมิลลิวินาที[cite: 5]
* **🧠 Smart Fallback Mechanism**: ลูปแผนสำรองอัจฉริยะที่ผู้ใช้จัดลำดับได้เอง เช่น คิวสลับสนาม คิวข้ามรอบเวลา หรือรายการคู่อันดับสำรองเจาะจง[cite: 5]
* **⏰ Low CPU 0.02s Live Waiting**: สแตนด์บายรีเฟรชหน้าต่างเบราว์เซอร์หน้างานทันทีที่นาฬิกาแตะเวลา 12:00:00 น. เป๊ะ โดยปรับค่า Sleep เป็น 0.02 วินาทีเพื่อไม่ให้กินทรัพยากรเครื่องแต่ยังคงความแม่นยำสูง[cite: 5]
* **🔒 Safe Harden Cleanup**: ครอบบล็อกคำสั่งนิรภัย (`finally:`) บังคับปิด Chrome และเคลียร์ RAM ออกจากระบบเสมอแม้สคริปต์จะทำงานล้มเหลวหรือถูกตัดสายอินพุตกลางคัน[cite: 5]

## 🗂️ ตรรกะการสลับคิวจองอัตโนมัติ (CASE 1 - 4)

บอตจะคำนวณและเข้าสู่ตรรกะการจองตามเงื่อนไขข้อมูลที่ผู้ใช้ป้อนใน Terminal[cite: 5]:

* **CASE 1 (อัตโนมัติทั้งหมด)**: ไม่ระบุทั้งสนามและเวลา บอตจะหาสนามแรกและช่วงเวลาแรกสุดที่เปิดว่างให้ทันที[cite: 5]
* **CASE 2 (ระบุสนามหลัก + คิวสำรอง)**: ระบุสนามเจาะจงแต่ไม่เลือกเวลา หากสนามหลักเต็ม บอตจะสลับไปเช็คสนามสำรองตามคิวตัวเลข 4 หลักที่คุณป้อนไว้ เช่น `6432`[cite: 5]
* **CASE 3 (ระบุเวลาหลัก + คิวสำรอง)**: ระบุรอบเวลาแต่ไม่เลือกสนาม หากรอบเวลานั้นเต็มหมดทุกคอร์ท บอตจะวนหาช่วงเวลาสำรองตามคิวที่ระบุข้ามสนามให้อัตโนมัติ[cite: 5]
* **CASE 4 (คู่อันดับสำรองสูงสุด)**: ระบุเจาะจงทั้งสนามและเวลา หากจับคู่แล้วไม่สำเร็จ สามารถป้อนแผนสำรองแบบจับคู่เฉพาะเจาะจงได้เรื่อย ๆ เช่น `62` (สนาม 6 เวลา 2) จนกว่าจะพิมพ์ `0` เพื่อบันทึก[cite: 5]
